    Base.css

    Классы-помощники

    Расширяют функциональность элементов.

    Ссылки

    .no-visited — у ссылки с данным класс убирает выделение :visited
    .pseudo — псевдоссылка, подчеркнута точками
    .no-border — ссылка без подчеркивания

    Если в ссылке находится картинка, подчеркивание убирается автоматически (ссылке добавляется класс .no-border), вернуть его можно добавив класс .border.

    Помощники

    [size] — размер: xsmall, small, medium, large или xlarge, от большего к меньшему.

    Размер Ширина экрана
    xsmall <= 420px
    small <= 720px
    medium <= 1080px
    large <= 1440px
    xlarge > 1440px

    .unselectable — запрещает выделение элемента и всех его потомков
    .clear — клеарфикс
    .text-left — выравнивание текста по левому краю
    .[size]-text-left — выравнивание текста по левому краю, адаптивный
    .text-center — выравнивание текста по центру
    .[size]-text-center — выравнивание текста по центру, адаптивный
    .text-right — выравнивание текста по правому краю
    .[size]-text-right — выравнивание текста по правому краю, адаптивный
    .uppercase — конвертация в прописные
    .lowercase — конвертация в строчные
    .capitalize — первые символы слов прописные
    .nowrap — запрет переноса строки
    .hyphens — автоматическая расстановка переносов (работает только в современных браузерах)
    .video — обертка для iframe, object и embed, пропорционально масштабирует видео
    .break-word — переносит непереносимые текстовые конструкции
    .visuallyhidden — скрывает блок специальным способом
    .hidden — присваивает блоку display: none
    .hidden-next — присваивает следующему блоку display: none
    .invisible — присваивает блоку visibility: hidden
    .table-responsive — горизонтальная прокрутка таблицы, если она не помещается
    .up — поднимает элемент над контекстом (добавляет ему z-index: 2)

    Экшн-классы

    .sync-hover — синхронное добавление класса .hover к ссылкам с одинаковым href
    .is-appeared — при появлении во viewport элементу присваиваются классы .appeared и .visible. При исчезновении из viewport класс .visible исчезает, а .appeared остается.

    Плавающие блоки

    Простые floating-блоки, можно указывать нужные параметры через css.

    Адаптивные блоки

    Автоматически подстраивающиеся под ширину строки блоки.

    Параметр data-rows указывает максимальное число строк, в которые должны помещаться блоки.

    Инлайн-блоки

    Простые инлайн-блоки, можно указывать нужные параметры через css.

    Адаптивные инлайн-блоки

    Автоматически подстраивающиеся под ширину строки инлайн-блоки.

    Параметр data-rows указывает максимальное число строк, в которые должны помещаться блоки.
    Поддерживаются теги menu и ul. Не нужно закрывать теги li.

    Сетка

    Автоматически подстраивающиеся под ширину строки адаптивные флексбокс-блоки. Весь принцип работы основан на флексбоксах, можно очень гибко манипулировать сеткой в CSS, если стандартных возможнонстей не хватает.

    В файле _base.scss можно увидеть настраиваемые параметры:
    $grid-columns: 12 — количество колонок в сетке
    $grid-gutters-large: 1.2vw — отступы внутри блоков на разных размерах экрана
    $grid-gutters-medium: 1.2vw
    $grid-gutters-small: 1.2vw

    [size] — размер: xsmall, small, medium, large или xlarge

    .modules — группа модулей
    .modules.-nearby — убирает отступы между колонками и под рядом
    .modules.-line — убирает отступ под рядом, оставляет между колонками
    .modules.[size]-row — модули в ряд, слева направо (по умолчанию)
    .modules.[size]-row-reverse — модули в ряд, справа налево
    .modules.[size]-column — модули в колонку, сверху вниз
    .modules.[size]-column-reverse — модули в колонку, снизу вверх
    .modules.[size]-wrap — переносимые модули
    .modules.[size]-wrap-reverse — переносимые модули справа налево
    .modules.[size]-nowrap — запрет переноса модулей
    .modules.[size]-start — модули по левому краю
    .modules.[size]-end — модули по правому краю
    .modules.[size]-centered — модули по центру
    .modules.[size]-space-between — одинаковое расстояние между модулями
    .modules.[size]-space-around — одинаковое расстояние вокруг модулей
    .modules.[size]-align-stretch — одинаковые по высоте (по умолчанию)
    .modules.[size]-align-centered — выравнивание по центру по вертиками

    .module — один модуль
    .module.[size]-[columns] — где columns это количество колонок, занимаемое модулем
    .module.[size]-[0..12] — автоматическая ширина модуля
    .module.[size]-auto — автоматическая ширина модуля
    .module.[size]-offset-[0..12] — отступ от предыдущей колонки
    .module.[size]-order-[0..12] — порядок сортировки модулей
    .module.ml-auto — сдвиг колонки вправо
    .module.[size]-ml-auto — сдвиг колонки вправо
    .module.mr-auto — сдвиг колонки влево
    .module.[size]-mr-auto — сдвиг колонки влево

    Адаптивность

    CSS

    Скрывает/показывает элементы при разных разрешениях экрана.

    .xlarge-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на большом экране
    .large-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на стандартном экране
    .medium-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на среднем экране
    .small-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на маленьком экране
    .xsmall-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на очень маленьком экране

    SASS

    Использование в SCSS

    @media only screen and (max-width: $screen-xlarge) {...}
    @media only screen and (max-width: $screen-large) {...}
    @media only screen and (max-width: $screen-medium) {...}
    @media only screen and (max-width: $screen-small) {...}
    @media only screen and (max-width: $screen-xsmall) {...}

    Сокращенный вариант:

    @include screen-xlarge {...}
    @include screen-large) {...}
    @include screen-medium) {...}
    @include screen-small) {...}
    @include screen-xsmall) {...}

    Кнопки

    Чтобы создать кнопку, надо добавить элементу класс .button.

    .button .button.-bold .button.-bordered
    .button.-large .button.-small .button.-xsmall

    if(Base.adaptive.small()){ ... }

    Примеры

    Анимации

    Запускает функцию с анимацией или управляет классом элемента при достижении им или его элементом-триггером определенной точки прокрутки.

    Инициализация

    TriggerID — указатель на ID эелемента, у которого мы отслеживаем положение
    Start — точка начала анимации (высота от верхнего края экрана в px или vh)
    End — точка окончания анимации (высота от верхнего края экрана в px или vh)
    FunctionName — название функции, выполнение которой начинается, когда элемент движется между Start и End
    Class — класс, присваиваемый ссылке, когда Target находится между Start и End
    ClassRemove — удалять ли Class у елемента после выхода его из зоны анимации, по умолчанию false

    Если элемент сам является триггером, TriggerID указывать не нужно.

    Функция FunctionName получает параметр со всеми полями элемента, описанными выше и параметром Progress, который изменяется от 0 до 1, в процессе прокрутки страницы и перемещения элемента.

    Пример

    Скроллер

    Прокручивает страницу до элемента с указанным ID.
    Автоматически применяется к элементам с data-scroll.

    Инициализация

    Target — ID цели
    Offset — смещение в пикселях (по умолчанию 0)
    Speed — скорость прокрутки (по умолчанию 500 мс)
    ClassActive — класс, присваиваемый ссылке, когда Target находится в зоне viewport

    API

    Base.scrollTo(target, offset, speed)

    target — ID цели
    offset — скорость прокрутки (по умолчанию 500 мс, необязательные параметр)
    speed — смещение в пикселях (по умолчанию 0, необязательные параметр)

    Загрузчик

    Запускает функцию подгрузки данных при прокрутке до конца блока.
    Автоматически применяется к элементам с data-loadmore.

    Инициализация

    FunctionName — название функции, выполняющей загрузку
    Offset — смещение в пикселях от нижнего края страница (по умолчанию 0)

    В функцию при каждом ее запуске передается инкрементный параметр page.

    Ленивая загрузка

    Загрузка изображений при их появлении в viewport.
    Работает с помощью метода IntersectionObserver, только в современных браузерах.
    Если IntersectionObserver не поддерживается, картинки загружаются обычным способом.
