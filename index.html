    CSS-классы

    Расширяют функциональность элементов.

    Ссылки

    .no-visited — у ссылки, обрамленной в данный класс, отстуствует выделение :visited
    .pseudo — псевдоссылка, подчеркнута точками
    .no-border — ссылка без бордюра

    Если в ссылке находится картинка, бордюр автоматически убирается, вернуть его можно добавив класс .border.

    Хэлперы

    .unselectable — запрещает выделение элемента
    .clear — clearfix
    .float-left
    .float-right
    .text-left
    .text-center
    .text-right
    .text-uppercase
    .text-lowercase
    .text-capitalize
    .nowrap — запрет переноса строки
    .hyphens — автоматическая расстановка переносов (работает только в современных браузерах)
    .video — пропорционально масштабирует видео
    .break-word — переносит непереносимые текстовые конструкции
    .visuallyhidden — скрывает блок
    .hidden — присваивает блоку display: none
    .hidden-next — присваивает следующему блоку display: none
    .invisible — присваивает блоку visibility: hidden
    .table-responsive — горизонтальная прокрутка таблицы, если она не помещается
    .up — поднимает элемент над контекстом

    Экшн-классы

    .double-hover — синхронное добавление класса .hover к ссылкам с одинаковым href
    .is-appeared — при появлении во viewport элементу присваиваются классы .appeared и .visible
    при исчезновении из viewport класс .visible исчезает, а .appeared остается.

    Простые блоки

    Простые float-блоки, можно указывать нужные параметры через css.

    <div class="blocks"> <div class="block"><img src="http://cms.loc/i/image.jpg"/></div> <div class="block"><img src="http://cms.loc/i/image.jpg"/></div> ... </div>

    Адаптивные блоки

    Автоматически подстраивающиеся под ширину строки блоки.

    Параметр data-rows указывает максимальное число строк, в которые должны помещаться блоки.

    <div class="blocks blocks-flex" data-rows="2"> <div class="block"><img src="http://cms.loc/i/image.jpg"/></div> <div class="block"><img src="http://cms.loc/i/image.jpg"/></div> ... </div>

    Простые инлайн-блоки

    Простые инлайн-блоки, можно указывать нужные параметры через css.

    <ul class="inline-blocks"> <li><img src="http://cms.loc/i/image.jpg"/> <li><img src="http://cms.loc/i/image.jpg"/> ... </ul>

    Адаптивные инлайн-блоки

    Автоматически подстраивающиеся под ширину строки инлайн-блоки.

    Параметр data-rows указывает максимальное число строк, в которые должны помещаться блоки.
    Поддерживаются теги menu и ul. Не нужно закрывать теги li.

    <ul class="inline-blocks inline-blocks-flex" data-rows="2"> <li><img src="http://cms.loc/i/image.jpg"/> <li><img src="http://cms.loc/i/image.jpg"/> ... </ul>

    .modules.[xsmall/small/medium/large/xlarge]-direction-column-reverse — распологает модули вертикально в обратном порядке, адаптивный

    .modules.[xsmall/small/medium/large/xlarge]-nowrap — делает блоки непереносимыми, адаптивный

    .modules.[xsmall/small/medium/large/xlarge]-wrap — делает блоки переносимыми, адаптивный
    .modules.wrap-reverse — делает блоки переносимыми в обратном порядке
    .modules.[xsmall/small/medium/large/xlarge]-wrap-reverse — делает блоки переносимыми в обратном порядке, адаптивный

    Адаптивность

    CSS

    Скрывает/показывает элементы при разных разрешениях экрана.

    .xlarge-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на большом экране
    .large-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на стандартном экране
    .medium-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на среднем экране
    .small-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на маленьком экране
    .xsmall-[hide|show|show-inline|show-inlineblock] — скрывает/показывает блок на очень маленьком экране

    SASS

    Использование в SCSS

    @media only screen and (max-width: $screen-xlarge) {...} — >= 1441px @media only screen and (max-width: $screen-large) {...} — >= 1440px @media only screen and (max-width: $screen-medium) {...} — >= 1080px @media only screen and (max-width: $screen-small) {...} — >= 720px @media only screen and (max-width: $screen-xsmall) {...} — >= 420px
    @include screen-xlarge {...} — >= 1441px @include screen-large) {...} — >= 1440px @include screen-medium) {...} — >= 1080px @include screen-small) {...} — >= 720px @include screen-xsmall) {...} — >= 420px

    Использование в JS

    if(Base.adaptive.xlarge){ ... } if(Base.adaptive.large){ ... } if(Base.adaptive.medium){ ... } if(Base.adaptive.small){ ... } if(Base.adaptive.xsmall){ ... }

    Анимации

    Запускает функцию с анимацией или управляет классом элемента при достижении им или его элементом-триггером определенной точки прокрутки.

    <div class="element" data-animate=' { "TriggerID": "element-trigger", "Start": "120vh", "End": "0vh", "FunctionName": "coverOut", "Class": "fixed", "ClassRemove": true } '>...</div>

    TriggerID — указатель на ID эелемента, у которого мы отслеживаем положение
    Start — точка начала анимации (высота от верхнего края экрана в px или vh)
    End — точка окончания анимации (высота от верхнего края экрана в px или vh)
    FunctionName — название функции, выполнение которой начинается, когда элемент движется между Start и End
    Class — класс, присваиваемый ссылке, когда Target находится между Start и End
    ClassRemove — удалять ли Class у елемента после выхода его из зоны анимации, по умолчанию false

    Если элемент сам является триггером, TriggerID указывать не нужно.

    Функция FunctionName получает параметр со всеми полями элемента, описанными выше и параметром Progress, который изменяется от 0 до 1, в процессе прокрутки страницы и перемещения элемента.

    Скроллер

    Прокручивает страницу до элемента с указанным ID.
    Автоматически применяется к элементам с data-scroll.

    <div id="top">...</div> <a data-scrollto="top">Наверх</a> // или с настройками <a data-scrollto='{"Target": "top", "Offset": "100", "Speed": "1000", "ClassActive": "active"}'>Наверх</a>

    Target — указатель на ID
    Speed — скорость прокрутки (по умолчанию 500 мс)
    Offset — смещение в пикселях (по умолчанию 0)
    ClassActive — класс, присваиваемый ссылке, когда Target находится в зоне viewport

    JS API

    Base.scrollTo(target, offset, speed)

    Наверх
    <a class="button" data-scrollto="top">Наверх</a>

    Lazyload

    Загрузка изображений при их появлении в viewport
    Работает с помощью метода IntersectionObserver, только в современных браузерах.
    Если IntersectionObserver не поддерживается, картинки загружаются обычным способом.

    <img src="http://cms.loc/base/alpha.png" data-src="image.jpg" data-srcset="..."/>

    AJAX

    Библиотека для работы с AJAX-запросами.

    Исходная документация: https://github.com/fdaciuk/ajax/blob/dev/README.md

    Узнать время

    Общий код

    var request = Base.ajax({ method: 'options', — здесь можно указать нужный метод url: '/api/users', data: { user: 'john' } }); request.then(function (response) {...})

    GET

    Base.ajax().get('/api/users', { id: 1 })

    POST

    Base.ajax().post('/api/users', { slug: 'john' }); // Можно использовать заголовки var request = Base.ajax({ headers: { 'x-access-token': '123@abc' } }); request.post('/login', { username: 'user', password: 'b4d45' })

    PUT

    Base.ajax().put('/api/users', { slug: 'john', age: 37 })

    DELETE

    Base.ajax().delete('/api/users', { id: 1 })

    Обработка ответов

    Base.ajax().get('/api/users').then(function (response, xhr) { // Код при успешном выполнении запроса }); Base.ajax().post('/api/users', { slug: 'john' }).catch(function (response, xhr) { // Код при ошибке }); Base.ajax().post('/api/users', { slug: 'john' }).always(function (response, xhr) { // Выполняется всегда, независимо от результата запроса })

    Отмена запроса

    const getLazyUser = Base.ajax().get('/api/users/lazy'); const timer = setTimeout(function () { getLazyUser.abort() }, 3000); getLazyUser.then(function (response) { clearTimeout(timer) console.log(response) })
